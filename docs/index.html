// 01 — the problem

agents are powerful.
they're also unguarded.

give an LLM access to your stripe keys, your database, your infrastructure — and you're one prompt away from a $10,000 bill or a dropped production table. fluffy sits between the agent and the world, enforcing rules you set before anything goes wrong. 

────────────────────────────────────────────────────────────────────────────────

// 02 — four guards, one pipeline

what fluffy protects.

🔐

secret redaction

register secrets once. agents and transcripts only ever see {{secret:name}} handles. real values are substituted at the last moment before execution and scrubbed from results, logs, audit trails, and exception traces.

two-layer: known-value + pattern scrub (luhn cards, api keys, high-entropy tokens)

💳

hard spend caps

per-use and daily caps per payment card, enforced at the database level with atomic check-and-reserve transactions. concurrent over-cap spending is structurally impossible — two racing $15 calls against a $25 daily cap: exactly one succeeds, every time.

default $25/day · timezone-aware windows · crash-orphaned reservations auto-expire

⚠️

confirmation gate

destructive actions raise a unique challenge with a random nonce phrase (DELETE 42). the agent must get human confirmation before proceeding. three wrong guesses voids the challenge. whitelist with one click for tools you trust.

5-minute expiry · single-use · undeclared destructive tools error at wrap time

🔑

permission broker

agents request elevated access mid-conversation — budget increases, tool grants. an approver chain (console, guardian bot, or custom UI) decides. approved grants take effect immediately, in the same session, no restart.

pluggable approver protocol · guardian bot auto-approves under configurable threshold

// 03 — five minutes

quick start.

install

# zero runtime dependencies — stdlib only
$ pip install fluffy

wrap a tool

from fluffy import Guard, ToolMeta, SpendSpec

guard = Guard()

# tag a tool as spend-tracked — $25/day cap by default
safe_charge = guard.wrap(
    stripe.charge,
    meta=ToolMeta(
        name="stripe.charge",
        tags={"spend"},
        spend=SpendSpec(
            card_id="ops",
            amount_from=lambda args, kw: kw["amount_cents"]
        )
    )
)

see a block happen

# $50 charge against $25 daily cap
>>> safe_charge(amount_cents=5000)
SpendLimitExceeded: Blocked: $50.00 requested, daily cap $25.00,
$0.00 already spent today. Remaining: $25.00

the block message is plain english — your agent can relay it to the user verbatim, or call guard.request_permission() to ask for a cap raise mid-conversation.

// 04 — architecture

how it works.

fluffy is a library, not a proxy. you import it, wrap the tools your agent calls, and the interceptor pipeline handles the rest. 

 agent calls safe_charge(amount_cents=5000)
          │
          ▼
┌─────────────────────────────────────────────┐
│ PermissionInterceptor   ← restricted tools denied here
│ SpendInterceptor        ← caps checked, money reserved atomically
│ ConfirmInterceptor      ← destructive? raise challenge
│ SecretResolveInterceptor ← {{secret:key}} → real value
├─────────────────────────────────────────────┤
│           TOOL EXECUTES                     │
├─────────────────────────────────────────────┤
│ SecretRedactInterceptor  ← scrub results
│ AuditInterceptor         ← write redacted row to sqlite
└─────────────────────────────────────────────┘
          │
          ▼
    result returned (or Blocked exception raised)

untagged calls skip all I/O — a set-intersection check, then straight to execution. only tagged calls pay for the pipeline. under 1 ms overhead for the common path.

// 05 — honesty

what fluffy does not defend against.

  • ✗ a malicious host process. fluffy runs inside your agent's Python process — if that process is compromised, no library can help.
  • ✗ prompt-injected confirm() calls. the agent itself calls guard.confirm(). if an attacker controls the agent's output, they can confirm their own challenges. confirmation must come from the human channel.
  • ✗ side channels outside wrapped tools. if the agent has unrestricted access to subprocess.run() or an unwrapped HTTP client, fluffy can't see those calls.

fluffy is one layer in a defense-in-depth strategy — not a silver bullet. pair it with sandboxing, least-privilege IAM, and human-in-the-loop for high-stakes actions.
